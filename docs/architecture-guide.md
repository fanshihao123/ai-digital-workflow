└──────┬───────┘         │ 调用

**调用关系**：`commands → steps → lib`，单向依赖，不反向调用。

## 4. 状态机：feature 生命周期

v4 用 `specs/{feature}/state.json` 统一管理状态，替代散落的 json 文件。

```
                        /start-workflow
                              │
                              ▼
                    ┌──────────────────┐
          ┌────────│      idle         │◄──────────────────┐
          │        └────────┬─────────┘                    │
          │     start       │                              │
          │                 ▼                              │
          │        ┌──────────────────┐        done        │
          │   ┌───►│    running       │───────────────────►│
          │   │    └──┬────┬────┬────┘                    │
          │   │       │    │    │                    ┌─────┴─────┐
          │   │       │    │    │                    │   done     │
          │   │       │    │    └──── fail ─────►┌───┴──────────┐│
          │   │       │    │                     │   failed     ││
          │   │       │    │                     └───┬──────────┘│
          │   │       │    │              resume     │           │
          │   │       │    │         ┌──────────────►┘           │
          │   │       │    │         │                           │
          │   │       │    ├── pause ──►┌──────────────┐        │
          │   │       │    │            │   paused      │        │
          │   │       │    │            └───┬───────────┘        │
          │   │       │    │     restart/   │                    │
          │   │       │    │      resume    │                    │
          │   │       │    │         ┌──────┘                    │
          │   │       │    │         │                           │
          │   │       │    │         ▼                           │
          │   │  answer│   │    (回到 running)                    │
          │   │       │    │                                     │
          │   │       ▼    │                                     │
          │   │  ┌─────────────────┐                             │
          │   │  │awaiting-answer  │  ── /answer ──► running     │
          │   │  └─────────────────┘                             │
          │   │                                                  │
          │   │       await-fix                                  │
          │   │       ▼                                          │
          │   │  ┌─────────────────┐                             │
          │   └──│ awaiting-fix    │  ── /fix-spec ──► running   │
          │      └─────────────────┘                             │
          │                                                      │
          └──────────────────────────────────────────────────────┘
                        (done 可以重新 start)
```

**合法状态转移表**：

| 当前状态 | 命令 | 目标状态 |
|---------|------|---------|
| idle | start | running |
| running | step | running |
| running | pause | paused |
| running | fail | failed |
| running | done | done |
| running | await-answer | awaiting-answer |
| running | await-fix | awaiting-fix |
| awaiting-answer | answer | running |
| awaiting-fix | fix-spec | running |
| paused | restart | running |
| paused | resume | running |
| failed | resume | running |
| done | start | running |

│  └── /pause → 改 requirements.md → /restart                     │

│                                                                  │

│                                                                  │

│                                                                  │

│                                                                  │
│  不知道现在什么状态？                                               │
